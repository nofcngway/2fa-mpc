**Gap:** Plan 02 behavior spec named TestSetup_SecretZeroized as a required test. The test is absent. The code correctly implements `defer crypto.Zeroize(raw)` but there is no test proving the raw bytes are zeroed after Setup returns.

**Problem:** `totp.GenerateSecret()` is called directly in `setup.go:44`. The `raw` variable is local to `Setup`. There is no way for the test (in `_test` package) to capture the raw slice pointer to assert zeroing.

**Solution:** Add a package-level function variable `generateSecretFunc` in `setup.go` that defaults to `totp.GenerateSecret`. This allows tests to inject a custom generator that captures the raw slice reference. This is a standard Go testing seam pattern (used widely in stdlib and production Go code).

**Step 1 — Modify `twofa/internal/services/twofaService/setup.go`:**

Add a package-level function variable near the top of the file (after imports, before ErrAlreadyEnabled):

```go
// generateSecretFunc is the function used to generate TOTP secrets.
// Package-level variable allows test injection for zeroization verification.
var generateSecretFunc = totp.GenerateSecret
```

Then change line 44 from:
```go
raw, base32Secret, err := totp.GenerateSecret()
```
to:
```go
raw, base32Secret, err := generateSecretFunc()
```

Leave everything else in setup.go unchanged. The `defer crypto.Zeroize(raw)` on line 48 stays as-is.

**Step 2 — Add TestSetup_SecretZeroized to `twofa/internal/services/twofaService/setup_test.go`:**

Add the following test function. Place it after the existing TestSetup_SharesZeroized test (around line 483). The test must:
1. Replace `generateSecretFunc` with a custom function that returns a known secret and captures the raw slice pointer
2. Run Setup with all mocks succeeding (same pattern as TestSetup_Success)
3. After Setup returns, assert every byte in the captured raw slice is zero
4. Restore the original `generateSecretFunc` in a defer

```go
func TestSetup_SecretZeroized(t *testing.T) {
    s := newSetupSuite(t)

    // Capture the raw secret slice to verify zeroization after Setup
    var capturedRaw []byte
    originalGenerate := twofaService.GenerateSecretFunc
    twofaService.GenerateSecretFunc = func() ([]byte, string, error) {
        raw, base32, err := originalGenerate()
        capturedRaw = raw
        return raw, base32, err
    }
    defer func() { twofaService.GenerateSecretFunc = originalGenerate }()

    s.storage.GetTwoFARecordMock.Expect(minimock.AnyContext, "test-user-id").Return(nil, nil)
    s.storage.CreateTwoFARecordMock.Expect(minimock.AnyContext, "test-user-id").Return(nil)
    s.storage.StoreBatchBackupCodesMock.Set(func(_ context.Context, _ string, _ []string) error {
        return nil
    })

    for i := 0; i < 3; i++ {
        s.mpcClients[i].StoreShareMock.Set(func(_ context.Context, _ *mpc_api.StoreShareRequest, _ ...grpc.CallOption) (*mpc_api.StoreShareResponse, error) {
            return &mpc_api.StoreShareResponse{ShareId: "share-id"}, nil
        })
    }

    _, _, err := s.service.Setup(context.Background(), "test-user-id", "user@example.com")

    assert.NilError(t, err)
    assert.Assert(t, capturedRaw != nil, "capturedRaw should not be nil")
    assert.Assert(t, len(capturedRaw) == 20, "TOTP secret should be 20 bytes, got %d", len(capturedRaw))
    for i, b := range capturedRaw {
        assert.Equal(t, byte(0), b, "raw secret byte at index %d should be zero after Setup, got %d", i, b)
    }
}
```

**IMPORTANT:** The variable must be exported as `GenerateSecretFunc` since the test file is in `twofaService_test` package (external test package). Update the variable declaration in setup.go accordingly:

```go
// GenerateSecretFunc is the function used to generate TOTP secrets.
// Exported to allow test injection for zeroization verification.

```

And the call site becomes:
```go
raw, base32Secret, err := GenerateSecretFunc()
```

This is safe because `GenerateSecretFunc` is only reassigned in tests, never in production code.
