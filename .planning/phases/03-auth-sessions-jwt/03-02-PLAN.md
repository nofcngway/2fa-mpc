<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> Login | Email + password cross here; must not leak which is wrong |
| client -> RefreshToken | Refresh JWT string crosses here; may be stolen/replayed |
| client -> ValidateToken | Access JWT string crosses here for validation |
| Auth service -> Redis | Token metadata stored/retrieved |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Spoofing | login.go | mitigate | Return same `ErrInvalidCredentials` for both non-existent user and wrong password -- prevents credential enumeration |
| T-03-07 | Spoofing | refresh_token.go | mitigate | Token family theft detection: valid JWT + missing JTI = stolen token -> revoke entire family via `DeleteTokenFamily` (D-08, D-09) |
| T-03-08 | Repudiation | refresh_token.go | mitigate | Token rotation: old JTI deleted before new one issued, creating audit trail of token lineage within family |
| T-03-09 | Information Disclosure | validate_token.go | mitigate | Returns only user_id + email from token claims (D-18) -- no privilege escalation via token payload |
</threat_model>
