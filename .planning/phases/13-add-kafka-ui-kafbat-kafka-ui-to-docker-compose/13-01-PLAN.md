  <name>Task 1: Add kafka-ui to root docker-compose.yml</name>
  <files>docker-compose.yml</files>
  <read_first>docker-compose.yml</read_first>
  <action>
Add a `kafka-ui` service to the root `docker-compose.yml` in the `# --- Infrastructure ---` section, after the `redis` service and before the `# --- Application Services ---` comment. Per D-01, D-02, D-03, D-05:

```yaml
  kafka-ui:
    image: kafbat/kafka-ui:latest
    depends_on:
      kafka:
        condition: service_started
    environment:
      KAFKA_CLUSTERS_0_NAME: mpc-2fa
      KAFKA_CLUSTERS_0_BOOTSTRAPSERVERS: kafka:9092
      DYNAMIC_CONFIG_ENABLED: "true"
    ports:
      - "8090:8080"
```

Key details:
- Image: `kafbat/kafka-ui:latest` (per D-01)
- Port mapping: `8090:8080` (per D-02, container listens on 8080 internally)
- No authentication configured (per D-03)
- Cluster name `mpc-2fa` for the shared Kafka instance
- `DYNAMIC_CONFIG_ENABLED: "true"` for runtime flexibility
- `depends_on` with `condition: service_started` matching existing Kafka dependency pattern
- All 3 topics (auth-events, twofa-events, mpc-events) are auto-discovered by kafka-ui from the connected broker (per D-05) -- no per-topic config needed
  </action>
