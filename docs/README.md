# Project Documentation

Глубокая документация по проекту MPC-2FA: архитектурные решения, протоколы безопасности, лог изменений. Эта папка — single source of truth для не-кодовой документации.

## Навигация

### 01 — Архитектура
*в работе* — высокоуровневые потоки данных, диаграммы взаимодействия сервисов.

### 02 — Сервисы
Карточки сервисов с RPC-методами, зависимостями, security-привязками. *В работе:* Auth, TwoFA, MPC Node, Gateway. Пока актуальная информация по сервисам — в их собственных README:
- [`auth/README.md`](../auth/README.md)
- [`twofa/README.md`](../twofa/README.md)
- [`mpc/README.md`](../mpc/README.md)
- [`gateway/README.md`](../gateway/README.md)

### 03 — Безопасность
- [MPC Fault Tolerance](03%20-%20Security/MPC%20Fault%20Tolerance.md) — модель отказоустойчивости 2-of-3, поведение по флоу при падении нод, тестовое покрытие
- [mTLS](03%20-%20Security/mTLS.md) — взаимная аутентификация всех internal gRPC-каналов, PKI, TLS 1.3, инструкции по добавлению нового сервиса

*В работе:* Shamir Secret Sharing, AES-256-GCM, JWT RS256, TOTP, Password Validation — отдельные deep-dive заметки. Пока контекст распределён в README сервисов и в ADR Log.

### 04 — Архитектурные решения (ADR)
- [ADR Log](04%20-%20Decisions/ADR%20Log.md) — все принятые архитектурные решения с обоснованием:
  - ADR-001 — Shamir для распределённого хранения секретов
  - ADR-002 — Собственная реализация Shamir в GF(256)
  - ADR-003 — AES-256-GCM для шифрования долей at-rest
  - ADR-004 — RS256 JWT (асимметричная подпись)
  - ADR-011 — mTLS между всеми internal сервисами

## Соглашения

- **Один топик — одна заметка.** Не смешивать темы.
- **Язык:** русский для пояснений, английский для кода и имён сущностей.
- **ADR** добавляются при принятии нетривиального архитектурного решения. Каждый ADR — дата, статус, контекст, решение, причина, последствия.
- **Changelog** обновляется после каждого значимого изменения.
