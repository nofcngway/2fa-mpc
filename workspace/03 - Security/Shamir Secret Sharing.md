# Shamir Secret Sharing

## Схема
- **Threshold**: 2-of-3 (любые 2 доли восстанавливают секрет, 1 доля — бесполезна)
- **Поле**: GF(256) — конечное поле из 256 элементов

## Алгоритм

### Split
1. Для каждого байта секрета:
   - Выбрать случайный коэффициент `a1` (crypto/rand)
   - Построить полином `f(x) = secret_byte + a1 * x` (степень = threshold - 1 = 1)
   - Вычислить `f(1)`, `f(2)`, `f(3)` — это доли
2. Результат: 3 доли, каждая — массив байт той же длины, что и секрет

### Combine (интерполяция Лагранжа)
1. Взять любые 2 доли с их индексами (x-координатами)
2. Для каждого байта:
   - Вычислить базисные полиномы Лагранжа в GF(256)
   - Восстановить `f(0)` = исходный байт секрета

### Арифметика GF(256)
- Сложение: XOR
- Умножение: через таблицы log/exp (генерирующий полином: x^8 + x^4 + x^3 + x + 1 = 0x11B)
- Деление: `a / b = exp[log[a] - log[b]]`
- Обратный элемент: `inv(a) = exp[255 - log[a]]`

## Тесты
- Split → Combine возвращает оригинал
- Любые 2 из 3 долей → корректное восстановление
- 1 из 3 долей → НЕ восстанавливает (нет достаточно данных)
- Пустой секрет
- Секрет длиной 1 байт
- Длинный секрет (20 байт = TOTP secret)
