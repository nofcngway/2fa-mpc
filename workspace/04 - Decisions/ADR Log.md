# Лог архитектурных решений (ADR)

## ADR-001: Shamir Secret Sharing вместо прямого хранения TOTP
- **Дата**: 2026-04-11
- **Решение**: TOTP-секрет разделяется на 3 доли (threshold=2), каждая хранится в отдельной MPC-ноде
- **Причина**: Компрометация одной ноды не раскрывает секрет. Соответствует теме  — безопасные многосторонние вычисления
- **Последствия**: Усложнение верификации (нужен запрос к 2 нодам), но значительное повышение безопасности

## ADR-002: Реализация Shamir с нуля (без библиотек)
- **Дата**: 2026-04-11
- **Решение**: Собственная реализация в GF(256) вместо сторонних библиотек
- **Причина**: Требование  — демонстрация понимания протокола. Контроль над криптографическими операциями
- **Последствия**: Необходимы тщательные unit-тесты, верификация корректности арифметики GF(256)

## ADR-003: AES-256-GCM для шифрования долей at-rest
- **Дата**: 2026-04-11
- **Решение**: Каждая MPC-нода шифрует свою долю перед сохранением в PostgreSQL
- **Причина**: Defense in depth — даже при доступе к БД ноды, доля остается зашифрованной
- **Последствия**: Уникальный ключ шифрования для каждой ноды, nonce через crypto/rand

## ADR-004: RS256 для JWT вместо HS256
- **Дата**: 2026-04-11
- **Решение**: Асимметричная подпись JWT (RSA)
- **Причина**: Только Auth Service владеет приватным ключом. Gateway и другие сервисы проверяют подпись публичным ключом без доступа к секрету подписи
- **Последствия**: Нужна пара ключей RSA, распространение публичного ключа между сервисами

## ADR-005: Clean Architecture (medialog/students pattern)
- **Дата**: 2026-04-11
- **Решение**: Каждый сервис — отдельный Go-модуль с единой структурой: api/ → cmd/app/ → config/ → internal/{api, bootstrap, services, storage, models, pb}
- **Причина**: Единообразие, тестируемость, слабая связанность через интерфейсы
- **Последствия**: Bootstrap-слой для DI, интерфейсы для всех зависимостей

## ADR-007: Миграция yaml.v3 → yaml/v4 и модернизация Go 1.26.2
- **Дата**: 2026-04-12
- **Решение**: Все сервисы (auth, twofa) мигрированы на `go.yaml.in/yaml/v4` (`yaml.Load` вместо `yaml.Unmarshal`). Весь код модернизирован для использования фич Go 1.22-1.26: `for range N`, `t.Context()`, `any`, `clear()`, `errors.AsType[T]()`, `errors.Is()`, `omitzero`, удаление устаревших loop variable captures
- **Причина**: Использование идиоматического современного Go 1.26.2, повышение читаемости и безопасности
- **Последствия**: Минимальная Go-версия проекта — 1.26+ (из-за `errors.AsType`), код чище и идиоматичнее

## ADR-006: pgx вместо ORM
- **Дата**: 2026-04-11
- **Решение**: Прямая работа с PostgreSQL через pgx, без GORM и других ORM
- **Причина**: Полный контроль над SQL, производительность, прозрачность запросов
- **Последствия**: Больше SQL-кода, но предсказуемое поведение
